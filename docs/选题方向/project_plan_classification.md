# 项目推进流程：集成分类算法模型按需筛选

## 1. 项目概述
**选题：** 集成分类算法模型按需筛选
**目标：** 构建一个机器学习工具（智能体），能够加载数据集，自动训练多种集成分类算法（如随机森林、AdaBoost、GBDT），对比它们的性能（准确率、F1分数等），并为用户“筛选”出表现最好的模型进行预测。
**技术栈推荐：** Python, `scikit-learn` (核心机器学习库), `pandas` (数据处理), `matplotlib` (绘图)。

### 1.1 选题原因（为什么选这个作为“需训练”里的入门项？）
1.  **门槛适中：** 相比深度学习（LSTM/Yolo），传统机器学习算法对数学推导和代码实现的要求更低，且不需要昂贵的 GPU 算力，普通笔记本 CPU 即可秒级完成训练。
2.  **数据友好：** 经典的分类数据集（如鸢尾花、红酒分类、泰坦尼克号）非常小巧且无需复杂的预处理（不用像图像那样做标注），`scikit-learn` 甚至内置了这些数据，开箱即用。
3.  **库支持强大：** `scikit-learn` 提供了统一的接口（fit/predict），只需几行代码就能切换不同的复杂算法，非常适合新手理解“模型训练”的全流程。
4.  **符合题意：** 题目要求“按需筛选”，这正好对应机器学习中的“模型选择”与“评估对比”环节，逻辑清晰。

---

## 2. 阶段一：环境与数据准备 (第1-2天)

### 2.1 环境搭建
安装数据科学三剑客。
```bash
python -m venv venv
# 激活环境后
pip install scikit-learn pandas matplotlib seaborn
```

### 2.2 数据获取与探索
*   **任务：** 确定一个数据集。
*   **推荐：** `sklearn.datasets.load_iris()` (鸢尾花，极简) 或 `load_digits()` (手写数字，稍有难度)。
*   **代码实践：** 使用 Pandas 读取数据，查看前几行，检查是否有缺失值，了解特征（X）和标签（y）。

---

## 3. 阶段二：核心算法实现 (第3-4天)

### 3.1 模型训练脚本
编写脚本 `model_trainer.py`。
*   **划分数据集：** 使用 `train_test_split` 将数据分为训练集和测试集。
*   **引入算法：** 导入三个经典的集成算法：
    1.  `RandomForestClassifier` (随机森林)
    2.  `AdaBoostClassifier`
    3.  `GradientBoostingClassifier`
*   **训练循环：** 编写一个循环，依次使用 `.fit(X_train, y_train)` 训练这三个模型。

### 3.2 评估与筛选
*   **任务：** 评估每个模型在测试集上的表现。
*   **指标：** 主要看 **准确率 (Accuracy)**。
*   **输出：** 打印出类似表格的结果：
    ```text
    模型: 随机森林 | 准确率: 0.96
    模型: AdaBoost | 准确率: 0.94
    ...
    最优模型是: 随机森林
    ```
*   **保存模型：** 使用 `joblib` 保存那个表现最好的模型。

---

## 4. 阶段三：智能体/工具封装 (第5-6天)

### 4.1 智能体逻辑
封装一个类 `ClassificationAgent`。
*   **功能 1 (Train)：** 接收一个数据文件路径，自动跑一遍上面的流程，通过训练对比，告诉你哪个算法适合这个数据，并保存该模型。
*   **功能 2 (Predict)：** 加载刚才保存的最优模型，接收用户输入的一组新数据，返回分类结果。

### 4.2 可视化展示 (加分项)
使用 `matplotlib` 画一个简单的条形图，直观对比三个算法的准确率，保存为图片 `comparison.png` 展示给用户看。

## 5. 阶段四：界面与文档 (第7天)

### 5.1 简单的交互界面
可以做一个简单的命令行菜单，也可以沿用 Streamlit：
*   **上传：** 用户上传 CSV 数据文件。
*   **按钮：** 点击“开始训练与筛选”。
*   **结果：** 显示哪个算法赢了，显示准确率对比图。

编写 `README.md`，截图展示训练过程的准确率输出，解释为什么在某次实验中随机森林比 AdaBoost 好（简单解释即可，例如“随机森林抗过拟合能力强”）。
