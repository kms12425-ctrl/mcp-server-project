# 项目推进流程：聚合API智能体封装组合

## 1. 项目概述
**选题：** 选取任意聚合API进行智能体封装组合
**目标：** 创建一个具备特定功能的智能体（Agent），通过调用第三方聚合 API（如天气查询、新闻聚合、汇率转换等）来回答用户问题或执行任务。
**技术栈推荐：** Python, 用于 HTTP 请求的 `requests` 库, 简单的命令行交互或 Streamlit/Gradio 界面（可选）。

### 1.1 选题原因（为什么选这个最容易上手？）
1.  **无需模型训练：** 相比于分类、聚类或 Yolo 识别等选题，本项目不需要收集数据集、清洗数据或进行耗时的模型训练过程。
2.  **代码复杂度低：** 核心逻辑主要涉及 HTTP 请求调用和基础的逻辑判断，依赖库少，Python 的 `requests` 库即可完成核心功能。
3.  **成果直观且即时：** 调用 API（如天气、新闻）能立即获得真实数据反馈，交互性强，演示效果好。
4.  **资源丰富：** 互联网上有大量免费开放的聚合 API 接口可供选择，降低了从零开始构建服务的难度。

---

## 2. 阶段一：选题与准备 (第1-2天)

### 2.1 确定 API 服务
选择一个稳定且易于使用的聚合 API 平台。
*   **推荐平台：** 阿里云 API 市场, 聚合数据, RollToolsApi, 百度 API Store 等。
*   **推荐 API 类型（入门级）：**
    *   **天气查询**（输入城市，返回天气）
    *   **每日新闻**（获取当天的头条新闻）
    *   **垃圾分类查询**（输入物品，返回分类）
*   **任务：** 注册平台账号，申请 API Key，阅读 API 文档。

### 2.2 环境搭建
确保本地安装了 Python 和必要的库。
```bash
python -m venv venv
# Windows 激活虚拟环境: venv\Scripts\activate
# Mac/Linux 激活虚拟环境: source venv/bin/activate

pip install requests
```

---

## 3. 阶段二：核心功能开发 (第3-4天)

### 3.1 编写 API 调用函数
创建一个 Python 脚本（例如 `agent_tools.py`），封装 API 的调用逻辑。
*   **输入：** 用户的查询参数（如城市名）。
*   **处理：** 构造请求 URL 和 Headers，发送 HTTP GET/POST 请求。
*   **输出：** 解析 JSON 响应，提取关键信息（剔除冗余数据）。

### 3.2 错误处理
为 API 调用添加异常处理。
*   处理 API Key 失效、请求超时、网络断开等情况。
*   当 API 返回错误码时，给用户友好的提示。

---

## 4. 阶段三：智能体封装 (第5-6天)

### 4.1 定义智能体类
创建一个 `Agent` 类来管理与用户的交互。
*   **意图识别（简单版）：** 使用关键词匹配（例如：如果输入包含“天气”，则调用天气工具）。
*   **意图识别（进阶版 -如果有能力）：** 接入一个简易的 LLM (如 OpenAI API 或本地模型) 来判断用户意图。*注：基础难度建议先做关键词匹配。*

### 4.2 交互逻辑实现
实现一个主循环 `main.py`。
1.  等待用户输入。
2.  Agent 分析输入指令。
3.  Agent 调度对应的 API 工具函数。
4.  格式化输出结果并展示给用户。

---

### 5.1 用户界面 (GUI)


编写 `app.py`，只有几行代码即可生成输入框和结果展示区域。
